MINIMAL KANBAN 1.1
СПРАВОЧНИК КОМАНД И MCP-ИНСТРУМЕНТОВ ДЛЯ CHATGPT

Назначение:
Этот файл нужен ChatGPT как краткая карта доступных MCP-инструментов и их практического смысла.
Работать нужно только с текущей доской Minimal Kanban и только через её MCP-коннектор.

ОСНОВНОЙ СТАРТОВЫЙ ПОРЯДОК
1. ping_connector
   Проверяет, что MCP-коннектор вообще жив.
2. bootstrap_context
   Возвращает identity коннектора, resource_url, board_name и scope_rule.
3. get_runtime_status
   Используется, если есть сомнение, что локальный API, MCP или туннель недоступны.

ГЛАВНЫЕ READ-ИНСТРУМЕНТЫ
get_board_context
- Краткая структурная сводка по доске.
- Полезен для понимания числа колонок, карточек и режима vehicle autofill.

get_board_snapshot
- Машиночитаемый снимок доски.
- Полезен для подтверждения состояния после изменений.

get_gpt_wall
- Главный обзорный источник по доске.
- Через него видны карточки, их тексты, положение по колонкам и история/лента событий.
- Это основной способ быстро войти в курс дела по всей доске.

get_cards
- Список активных карточек.
- Удобен для обзорного списка и массовых операций.

get_card
- Полное чтение одной карточки.
- Используется после create/update/autofill как read-after-write проверка.

get_card_log
- История одной карточки.
- Нужен для понимания, что уже менялось и кем.

list_columns
- Возвращает колонки доски.

list_archived_cards
- Возвращает архивные карточки.

list_overdue_cards
- Возвращает карточки с просрочкой или горящими сроками.

search_cards
- Ищет карточки по vehicle, title, description, tags, short_id и смысловым маркерам.

ГЛАВНЫЕ WRITE-ИНСТРУМЕНТЫ
create_card
- Создание новой карточки.
- Можно передавать обычные поля карточки и vehicle_profile.

update_card
- Обновление существующей карточки.
- Должно аккуратно менять данные без silent overwrite ручных полей vehicle_profile.

move_card
- Перемещение одной карточки между колонками.

bulk_move_cards
- Массовое перемещение группы карточек.
- Используется вместо длинной серии move_card.

archive_card
- Архивирование карточки.

restore_card
- Восстановление карточки из архива.

create_column
- Создание нового столбца.

delete_column
- Удаление пустого столбца.
- Не использовать для занятых столбцов без проверки.

set_card_deadline
- Установка или обновление срока.

set_card_indicator
- Установка визуального индикатора/сигнала.

ИНСТРУМЕНТЫ РАБОТЫ С АВТОМОБИЛЕМ
autofill_vehicle_data
- Основной инструмент автозаполнения автомобиля.
- В версии 1.1 он работает по схеме card_content_first:
  сначала анализирует поля самой карточки:
  - vehicle
  - title
  - description
  - raw_text, если он есть
- Потом безопасно дополняет compact vehicle profile.

get_card и get_cards
- Возвращают не только полный vehicle_profile, но и compact vehicle profile.

АКТУАЛЬНЫЕ УПРОЩЁННЫЕ ПОЛЯ АВТОМОБИЛЯ В 1.1
- make_display
- model_display
- production_year
- vin
- engine_model
- gearbox_model
- drivetrain
- oem_notes

ПРАКТИЧЕСКИЕ ПРАВИЛА ДЛЯ CHATGPT
1. Для общего понимания доски сначала используй get_gpt_wall.
2. Для проверки подключения сначала используй ping_connector и bootstrap_context.
3. После изменения карточки обязательно делай get_card.
4. Для массового переноса используй bulk_move_cards, а не цепочку move_card.
5. Для автомобиля работай через compact vehicle profile 1.1.
6. Автозаполнение автомобиля должно опираться на содержимое самой карточки.
7. Если VIN, мотор или КПП не удалось надёжно определить, не выдумывать значения.

КРАТКО
Для старта:
ping_connector -> bootstrap_context -> get_gpt_wall

Для карточки:
search_cards/get_card -> update/create -> get_card

Для автомобиля:
заполнить card body -> autofill_vehicle_data -> get_card
