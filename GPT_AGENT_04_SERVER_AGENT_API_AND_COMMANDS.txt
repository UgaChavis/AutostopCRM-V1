Auth model
- Use operator session auth.
- Login route: POST /api/login_operator
- Send the returned token in X-Operator-Session.

Minimal login example
curl -sS -X POST http://127.0.0.1:8000/api/login_operator ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"admin\",\"password\":\"admin\"}"

Server commands
- deploy:
  cd /opt/autostopcrm
  ./deploy.sh
- restart containers:
  cd /opt/autostopcrm
  docker compose restart
- app logs:
  cd /opt/autostopcrm
  docker compose logs -f autostopcrm
- agent logs:
  cd /opt/autostopcrm
  docker compose logs -f autostopcrm-agent

Important operating rules
- Use the current board API as the only write surface for tasks.
- Do not enqueue duplicate scheduled runs for the same schedule when one is already active.
- For card autofill, enable the 4-hour mode only on the target card and let the scheduler handle repeats.
- If a repair order is unpaid, do not close it. Payment is required first.
