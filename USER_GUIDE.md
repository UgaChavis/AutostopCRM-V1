# Руководство пользователя

## Для кого эта инструкция

Это руководство для пользователя, который хочет:

- открыть программу без установки
- запускать её двойным щелчком
- работать без терминала и без команд
- создавать карточки и столбцы
- использовать автоматический обратный отсчёт
- понимать лампочку состояния
- сохранять данные между перезапусками

## Что важно знать сразу

Эта версия программы работает в portable-режиме.

Это значит:

- ничего устанавливать не нужно
- отдельный backend запускать не нужно
- отдельно запускать `python`, `npm`, `node` и другие команды не нужно
- все внутренние части стартуют автоматически

## Как открыть программу

1. Откройте папку `release`.
2. Найдите файл `Start Kanban.exe`.
3. Дважды щёлкните по нему.

После этого должно открыться окно программы.

## Как понять, что запуск прошёл успешно

После запуска вы увидите:

- тёмное окно программы
- заголовок `Минимальная канбан-доска`
- стартовые столбцы `Входящие`, `В работе`, `Готово`
- кнопку `Новая карточка`
- кнопку `Новый столбец`
- кнопку `Помощь`
- строку с адресом локального API

При первом запуске также откроется экран `Быстрый старт`.

## Что делать на первом запуске

1. Прочитайте шаги встроенной помощи.
2. Нажимайте `Далее`.
3. На последнем шаге нажмите `Готово`.
4. После этого вы попадёте на основную доску.

## Как создать карточку

1. Нажмите `Новая карточка`.
2. Заполните поле `Заголовок`.
3. При необходимости заполните поле `Описание`.
4. В блоке `Срок` укажите:
   `Дни`
   `Часы`
5. Нажмите `Сохранить`.

После этого карточка появится на доске.

## Как работает обратный отсчёт

После сохранения карточки отсчёт начинается автоматически.

На карточке вы будете видеть строку вида:

- `2д 04:12:33`

Это означает:

- `2д` — осталось 2 дня
- `04` — осталось 4 часа
- `12` — осталось 12 минут
- `33` — осталось 33 секунды

## Что означает лампочка состояния

На каждой карточке есть маленькая лампочка.

Она меняется автоматически:

- зелёная — времени достаточно
- жёлтая — срок близко
- красная — срок истёк

Когда срок истёк:

- лампочка становится красной
- карточка получает лёгкий акцент
- оставшееся время фиксируется на `0д 00:00:00`

## Как открыть и изменить карточку

Есть один основной способ:

1. Дважды щёлкните по карточке.

В окне карточки можно изменить:

- заголовок
- описание
- срок

После изменения нажмите `Сохранить`.

## Как архивировать карточку

1. Дважды щёлкните по карточке.
2. В окне карточки нажмите `В архив`.

После этого карточка исчезнет с активной доски, но останется в локальном хранилище и будет доступна через API.

## Как создать новый столбец

1. Нажмите кнопку `Новый столбец`.
2. Введите название столбца.
3. Нажмите `Сохранить`.

После этого:

- новый столбец сразу появится на доске
- он сохранится автоматически
- после закрытия и повторного запуска останется на месте

## Как перемещать карточки

1. Наведите курсор на карточку.
2. Зажмите левую кнопку мыши.
3. Перетащите карточку в нужный столбец.
4. Отпустите кнопку мыши.

Можно перемещать карточки и в стартовые, и в пользовательские столбцы.

## Как снова открыть помощь

1. Запустите программу.
2. Нажмите `Помощь`.
3. Используйте кнопки `Назад` и `Далее`.

## Как открыть настройки интеграции

1. Посмотрите в правый верхний угол главного окна.
2. Нажмите кнопку с шестерёнкой.
3. Откроется отдельное окно `Настройки интеграции`.

Секции внутри окна:

- `Общие`
- `Доступ и токены`
- `ChatGPT / OpenAI`
- `MCP`
- `Диагностика`

Что можно изменить:

- включение интеграции
- параметры локального API
- токены локального API и MCP
- параметры OpenAI-совместимого API
- параметры MCP
- автоподключение и тестовый режим

Кнопки внизу окна:

- `Применить` — сохранить без закрытия окна
- `Сохранить` — сохранить и закрыть окно
- `Отмена` — закрыть окно без сохранения текущих изменений
- `Сбросить по умолчанию` — вернуть значения по умолчанию после подтверждения
- `Проверить соединение` — проверить локальный API, MCP и OpenAI-совместимый API

## Как понять, что настройки сохранились

1. Откройте окно настроек.
2. Измените нужные поля.
3. Нажмите `Применить` или `Сохранить`.
4. Убедитесь, что внизу окна появилось сообщение об успешном сохранении.
5. Закройте окно.
6. Откройте его снова.
7. Проверьте, что значения остались на месте.

Важно:

- секретные поля скрыты по умолчанию
- кнопка `Показать` временно открывает секрет в поле
- кнопка `Копировать` копирует значение в буфер обмена
- изменения host, port и токенов начинают влиять на следующие запуски и внешние подключения; уже запущенный локальный API меняется после перезапуска программы

## Как использовать проверку соединения

1. Откройте окно настроек.
2. Заполните поля подключения.
3. Нажмите `Проверить соединение`.
4. Дождитесь итогового окна с результатом.
5. Проверьте секцию `Диагностика`.

В секции `Диагностика` видно:

- путь к `settings.json`
- текущий адрес API приложения
- время последней проверки
- отдельный статус локального API
- отдельный статус MCP
- отдельный статус OpenAI

## Как закрыть программу

1. Нажмите `X` в правом верхнем углу окна.

или

1. Закройте окно через панель задач Windows.

Ничего вручную сохранять не нужно.

## Что делать при типовых проблемах

### Программа не запускается

1. Убедитесь, что запускаете именно `Start Kanban.exe`.
2. Закройте другие открытые копии программы.
3. Попробуйте запустить файл ещё раз.
4. Если ошибка повторяется, откройте лог:
   `%APPDATA%\Minimal Kanban\logs\minimal-kanban.log`

### Не сохраняются изменения

Проверьте:

1. Есть ли файл `%APPDATA%\Minimal Kanban\state.json`.
2. Нет ли рядом файла `%APPDATA%\Minimal Kanban\state.corrupted.json`.
3. Нет ли записей об ошибках в логе.

### Не получается создать карточку

Проверьте:

1. Поле `Заголовок` не пустое.
2. Заголовок не длиннее 120 символов.
3. Описание не длиннее 5000 символов.
4. Срок больше нуля.

### Не получается создать столбец

Проверьте:

1. Название столбца не пустое.
2. Название не длиннее 40 символов.
3. Столбца с таким названием ещё нет.

### Ошибка в окне настроек интеграции

Проверьте:

1. Что поля `Хост`, `Порт`, `Базовый URL` и `Путь MCP` заполнены корректно.
2. Что порт указан числом в допустимом диапазоне.
3. Что `Ключ API OpenAI` заполнен, если вы проверяете OpenAI-совместимый API.
4. Что программа уже запущена, если вы проверяете локальный API.
5. Что MCP server уже поднят, если вы проверяете внешний MCP endpoint.

## Где лежат данные и логи

- данные: `%APPDATA%\Minimal Kanban\state.json`
- настройки интеграции: `%APPDATA%\Minimal Kanban\settings.json`
- логи: `%APPDATA%\Minimal Kanban\logs\minimal-kanban.log`

## Что важно помнить

- программа полностью локальная
- интернет для обычной работы не нужен
- локальный API стартует автоматически
- portable-файл можно запускать без установки в систему
