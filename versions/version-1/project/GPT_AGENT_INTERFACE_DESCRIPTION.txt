ИНТЕРФЕЙС MINIMAL KANBAN: КРАТКОЕ ОПИСАНИЕ

Это тёмная минималистичная канбан-доска в техногенном стиле, похожая на старый диспетчерский пульт или промышленный терминал.

ОБЩИЙ ВИД
- Весь фон очень тёмный, зелёно-графитовый, почти чёрный.
- В левом верхнем углу находится заголовок "КАНБАН / ПУЛЬТ".
- Под заголовком идёт короткая служебная строка в стиле терминала: минимальный интерфейс, полный журнал, хост в сети.
- Ниже расположена узкая статусная плашка со сводкой по состоянию доски: сколько карточек активно и сколько в архиве.

СТРУКТУРА ДОСКИ
- Основная часть экрана — широкое горизонтальное полотно с колонками.
- Колонки узкие, прямоугольные, тёмные, выстроены слева направо.
- Примеры колонок: ПРИЁМКА, ДИАГНОСТИКА, ЭЛЕКТРИКИ, СЛЕСАРЯ, КУЗОВ, ТОКАРКА, ЗАПЧАСТИ, СОГЛАСОВАНИЕ, ВЫДАЧА.
- В заголовке каждой колонки есть её название и маленький счётчик карточек.
- Внизу каждой колонки есть кнопка "+ КАРТОЧКА".

КАРТОЧКИ
- Карточки светлые, тёпло-бежевые, на фоне тёмных колонок выглядят как бумажные технологические бланки.
- У карточек тонкая контрастная рамка; у тревожных карточек есть жёлтая или красная подсветка.
- В верхней строке карточки обычно идут: МАРКА / КОРОТКИЙ ЗАГОЛОВОК.
- Ниже расположен плотный текст описания.
- В нижней части карточки размещены короткие служебные поля: сигнал, метки, файлы, журнал, остаток времени.
- Общая компоновка карточки компактная, плотная и функциональная.

ВЕРХНИЕ КНОПКИ
- Справа в верхней панели расположены быстрые кнопки управления.
- Обычно там есть: ОПЕРАТОР, АРХИВ, СТЕНА, + СТОЛБЕЦ, + КАРТОЧКА.
- Кнопка СТЕНА связана с полным текстовым контекстом доски для GPT.

СТИЛЬ
- Визуальный язык строгий, инженерный, без декоративной мягкости.
- Интерфейс выглядит как рабочая доска диспетчера автосервиса.
- Основной акцент не на красоте, а на читаемости, плотности и быстром обзоре состояния.

ВАЖНО ДЛЯ GPT-АГЕНТА
- Это не яркий современный task manager, а тёмный диспетчерский пульт.
- Основная навигационная логика: колонки слева направо, карточки внутри колонок сверху вниз.
- Визуально приоритетные элементы: название колонки, название карточки, описание, сигнал, метки, срок, журнал.
- СТЕНА — отдельный контекстный слой, который даёт полное понимание состояния доски и истории изменений.
