# Настройка MCP и GPT-агента

## Что есть в проекте

- `Local API` — локальный HTTP API самой доски. Он нужен приложению и MCP-слою.
- `MCP` — отдельный endpoint поверх `Local API`. Именно его нужно подключать к ChatGPT/GPT-агенту.
- `Settings -> Integration / GPT / MCP` — центральное окно, где уже видны все host, port, URL, токены, проверки и экспорт карточки подключения.

## Базовый сценарий подключения

1. Запустите программу.
2. Нажмите шестерёнку в правом верхнем углу.
3. Откройте раздел настроек интеграции.
4. Убедитесь, что:
   - включена интеграция;
   - включён `MCP`;
   - локальный API не отключён без необходимости.
5. Если нужен внешний доступ:
   - заполните `Public HTTPS Base URL`, либо
   - заполните `Tunnel URL`, либо
   - укажите `Full MCP URL override`.
6. При необходимости включите bearer-режим и сгенерируйте токен для MCP.
7. Нажмите:
   - `Запустить MCP сервер`;
   - `Проверить локальный API`;
   - `Проверить MCP`;
   - `Проверить внешний endpoint`;
   - при необходимости `Проверить OpenAI`.
8. Скопируйте `Итоговый MCP URL для GPT-агента`.
9. Если включён bearer-режим, скопируйте `Bearer token MCP`.
10. Подключите этот URL в ChatGPT как MCP endpoint.

## Как формируется итоговый MCP URL

Приоритет строгий:

1. `Full MCP URL override`
2. `Public HTTPS Base URL + mcp_path`
3. `Tunnel URL + mcp_path`
4. fallback на `Local MCP URL`

Важно:

- fallback на локальный URL годится только для локальной диагностики;
- для ChatGPT на телефоне нужен внешний `https://` адрес;
- `localhost` и `127.0.0.1` для удалённого подключения не подходят.

## Как вывести MCP наружу

### Вариант 1. HTTPS reverse proxy

1. Поднимите домен или поддомен.
2. Проксируйте внешний путь на локальный MCP runtime, например:
   - внешний `https://kanban.example.com/mcp`
   - внутренний `http://127.0.0.1:41831/mcp`
3. Укажите `Public HTTPS Base URL = https://kanban.example.com`.
4. Проверьте `Итоговый MCP URL для GPT-агента`.

### Вариант 2. Tunnel

1. Поднимите HTTPS tunnel на локальный MCP runtime.
2. Полученный адрес вставьте в `Tunnel URL`.
3. Оставьте `mcp_path` как есть, если tunnel уже прокидывает путь `/mcp`.
4. Нажмите `Проверить внешний endpoint`.

## Какой URL вставлять в ChatGPT

- Всегда вставляйте `Итоговый MCP URL для GPT-агента`.
- Не вставляйте `Local API URL` напрямую.
- Не вставляйте `Runtime URL локального API`.
- Не вставляйте `Local MCP URL`, если подключение идёт с телефона или из внешнего интернета.

## Как проверить руками

Минимальный набор:

1. `Проверить локальный API` должен вернуть `Успешно`.
2. `Проверить MCP` должен вернуть `Успешно`.
3. `Проверить внешний endpoint` должен вернуть `Успешно`, если вы используете внешний URL.
4. В `Экспорт и быстрые действия` можно:
   - скопировать карточку подключения;
   - экспортировать карточку подключения в `GPT_MCP_CONNECTION_CARD.txt`;
   - экспортировать текущие настройки интеграции.

## Типовые ошибки

- `MCP выключен`:
  включите MCP в настройках.
- `Итоговый MCP URL не начинается с https://`:
  для ChatGPT нужен внешний HTTPS URL.
- `Итоговый MCP URL указывает на localhost`:
  нужен внешний tunnel или reverse proxy.
- `Не удалось подключиться к адресу MCP ...`:
  MCP runtime не запущен, порт занят или tunnel/proxy не ведёт на нужный endpoint.
- `Для bearer-режима нужен токен`:
  сгенерируйте токен кнопкой в секции авторизации.
- `Изменены host, port, path или токены`:
  для уже поднятых процессов нужен перезапуск приложения или MCP runtime.

## Ограничения текущего этапа

- Секреты пока хранятся в `%APPDATA%\\Minimal Kanban\\settings.json` без secure storage.
- Полноценного OAuth provider пока нет.
- Уже запущенный локальный API не применяет новые host/port/token без перезапуска приложения.
- Уже запущенный MCP runtime не применяет новые host/port/path/token без остановки и повторного запуска.
