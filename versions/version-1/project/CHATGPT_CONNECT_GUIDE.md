# Подключение к ChatGPT

Этот документ показывает практический путь, как подключить текущую доску к ChatGPT на телефоне через MCP.

## Цель

После настройки вы сможете из ChatGPT отправлять команды вроде:

- создай карточку
- перемести карточку
- создай столбец
- покажи просроченные карточки

и эти команды будут менять вашу доску.

## Важное разделение

Текущая схема не использует управление мышью и не кликает по интерфейсу.

Используется нормальный путь:

```text
ChatGPT
  ->
remote MCP server
  ->
локальный API доски
  ->
backend доски
  ->
state.json / UI
```

## Что нужно на Windows-машине

1. Локальная доска или hidden backend.
2. Запущенный MCP server.
3. Временный или постоянный HTTPS URL до MCP server.

## Самый простой тестовый путь

### Шаг 1. Запустите MCP server

Откройте PowerShell в корне проекта и выполните:

```powershell
.\scripts\run_mcp_server.ps1
```

По умолчанию MCP server слушает:

```text
http://127.0.0.1:41831/mcp
```

### Шаг 2. Поднимите HTTPS tunnel

Для теста нужен публичный HTTPS URL, потому что ChatGPT на Android не сможет обратиться к вашему `127.0.0.1`.

Самый простой dev-вариант:

```powershell
cloudflared tunnel --url http://127.0.0.1:41831
```

После запуска вы получите публичный `https://...trycloudflare.com` URL.

Использовать нужно endpoint MCP:

```text
https://ВАШ-URL.trycloudflare.com/mcp
```

Важно:

- это dev/test путь
- для постоянного использования лучше выделить стабильный hostname и нормальный reverse proxy / managed tunnel
- если вы включили `MINIMAL_KANBAN_MCP_BEARER_TOKEN`, то для прямого подключения через ChatGPT app без OAuth это может быть неудобно; для первого теста проще использовать tunnel без bearer token и ограничить доступ короткоживущим URL

### Шаг 3. Подключите сервер в ChatGPT

По актуальным OpenAI документам путь для ChatGPT app / Developer mode такой:

1. Откройте настройки ChatGPT.
2. Импортируйте свой custom MCP server.
3. Укажите ваш публичный HTTPS URL вида:

```text
https://ВАШ-URL.trycloudflare.com/mcp
```

4. Дайте приложению понятное имя, например `Minimal Kanban`.
5. Проверьте, что ChatGPT видит инструменты:
   - `create_card`
   - `move_card`
   - `get_cards`
   - `create_column`

## Первый тест с телефона

После подключения из ChatGPT попробуйте подряд:

1. `Покажи мои столбцы на доске.`
2. `Создай карточку "Позвонить клиенту" в столбце inbox со сроком 0 дней и 4 часа.`
3. `Покажи все карточки.`
4. `Создай столбец "Ожидание ответа".`
5. `Перемести карточку "Позвонить клиенту" в столбец ожидание ответа.`
6. `Сделай этой карточке красный индикатор.`

Ожидаемый результат:

- столбец появляется в доске
- карточка создаётся
- карточка перемещается
- дедлайн и лампочка меняются

## Прямой тест через Responses API

Если вы хотите проверить интеграцию не через UI ChatGPT, а программно, используйте MCP tool через Responses API.

Пример лежит в файле:

- [mcp-tools-example.json](mcp-tools-example.json)

Смысл такой:

```json
{
  "model": "gpt-5",
  "input": "Создай карточку Позвонить клиенту на 4 часа",
  "tools": [
    {
      "type": "mcp",
      "server_label": "minimal_kanban",
      "server_url": "https://your-tunnel.example/mcp",
      "allowed_tools": [
        "create_card",
        "get_cards",
        "move_card"
      ],
      "require_approval": "never"
    }
  ]
}
```

## Безопасность

### Что уже есть в проекте

- ограниченный набор инструментов только для доски
- валидация входных данных
- bearer token для локального API
- bearer token для MCP server
- никаких произвольных команд на компьютере

### Что рекомендует OpenAI

По актуальным OpenAI docs:

- remote MCP servers можно подключать к ChatGPT и Responses API
- для публичных серверов рекомендуется OAuth и dynamic client registration

Ссылки:

- [MCP and connectors guide](https://developers.openai.com/api/docs/guides/tools-connectors-mcp)
- [Building MCP servers](https://developers.openai.com/api/docs/mcp)

### Практический вывод для текущего проекта

Для dev/test сейчас есть два реалистичных режима:

1. Быстрый режим:
   - без OAuth
   - MCP server на localhost
   - наружу отдаётся через временный HTTPS tunnel

2. Более строгий режим:
   - локальный API защищён bearer token
   - сам MCP server защищён bearer token
   - прямой доступ лучше использовать из собственного клиента или Responses API

Для production-grade ChatGPT app в следующем этапе нужно добавить OAuth provider.

## Если ChatGPT не видит сервер

Проверьте по порядку:

1. MCP server действительно запущен.
2. Локально открывается `http://127.0.0.1:41831/mcp`.
3. Tunnel действительно проксирует именно порт `41831`.
4. Наружный URL заканчивается на `/mcp`.
5. На Windows-машине не закрыт процесс `run_mcp_server.ps1`.
6. Если включён bearer token, клиент умеет его передавать.

## Ограничения текущего этапа

- полноценный OAuth flow ещё не добавлен
- для первого подключения в ChatGPT удобнее использовать dev tunnel
- основной безопасный production-шаг дальше: OAuth + постоянный HTTPS hostname + reverse proxy

## Как использовать новое окно настроек перед подключением

Перед тем как открывать доступ из ChatGPT, удобнее сначала заполнить окно `Настройки интеграции` в самом приложении.

Что стоит задать там заранее:

- включение интеграции
- параметры локального API
- bearer token локального API
- включение MCP
- host, port и path MCP
- публичный HTTPS-адрес или адрес туннеля
- bearer token MCP
- параметры OpenAI-совместимого API

Практический порядок:

1. Запустите `Start Kanban.exe`.
2. Нажмите шестерёнку в левом верхнем углу.
3. Заполните секции `Общие`, `Доступ и токены`, `ChatGPT / OpenAI` и `MCP`.
4. Нажмите `Проверить соединение`.
5. Убедитесь, что в секции `Диагностика` появились ожидаемые статусы.
6. Нажмите `Сохранить`.
7. После этого запускайте MCP server и открывайте туннель.

Важно:

- текущий уже запущенный локальный API не меняет host, port и токены "на лету"; эти параметры начинают влиять на следующие запуски
- `settings.json` хранится отдельно от карточек в `%APPDATA%\Minimal Kanban\settings.json`
- секреты пока не вынесены в системное secure storage, поэтому для публичного теста лучше использовать ограниченные токены и временные туннели
