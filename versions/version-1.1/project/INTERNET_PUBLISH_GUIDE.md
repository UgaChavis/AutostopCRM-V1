# Публикация Доски В Интернет

Эта версия `Minimal Kanban` уже подготовлена к удалённому доступу через браузер.

Что уже сделано в приложении:

- доска и API работают на одном сервере;
- веб-интерфейс умеет работать с bearer-токеном;
- есть готовая `ссылка доступа` для браузера;
- внешняя ссылка может быть отдельной от локального `http://127.0.0.1:41731`;
- GPT / MCP и браузерная доска теперь могут опираться на один и тот же внешний URL.

## Безопасный минимум

Для публикации в интернет рекомендуется такой базовый режим:

1. Включить `bearer` для локального API.
2. Сгенерировать отдельный `local_api_bearer_token`.
3. Поднять внешний HTTPS-адрес через tunnel, reverse proxy или VPS.
4. В поле `Внешний URL доски / API override` указать внешний корневой адрес доски.

Пример:

- локально: `http://127.0.0.1:41731`
- снаружи: `https://board.example`

Важно:

- в override для браузерной доски нужно указывать именно корень доски;
- если раньше использовался адрес вида `https://board.example/api`, приложение теперь корректно выведет из него корень доски для share-ссылки;
- сама доска открывается по корню, а API остаётся под `/api/*`.

## Как открыть доску с другого компьютера

После настройки в окне хоста появится:

- `Локальный адрес`
- `Сетевой адрес`
- `Ссылка доступа`

Для удалённого пользователя нужно копировать именно `Ссылку доступа`.

Пример:

`https://board.example?access_token=...`

Это уже готовая браузерная ссылка. Она:

- открывает саму доску;
- даёт веб-интерфейсу токен;
- позволяет работать с защищённым API и скачивать вложения.

## Что должен делать внешний прокси или tunnel

Наружный HTTPS-слой должен проксировать:

- `/` и все обычные браузерные запросы на локальную доску;
- `/api/*` на тот же локальный сервер.

Иными словами, внешний адрес должен смотреть на тот же экземпляр приложения, который держит доску на твоём ПК.

## Быстрый сценарий

1. Запусти [Start Kanban.exe](C:/Users/User/Desktop/Codex/minimal-kanban/release/Start%20Kanban.exe).
2. Открой `Настройки GPT / MCP`.
3. В секции `Локальный API` включи `bearer`.
4. Сгенерируй токен.
5. Укажи `Внешний URL доски / API override`.
6. Сохрани настройки.
7. Скопируй из окна хоста `Ссылку доступа`.
8. Открой её с другого компьютера.

## Что проверить перед реальной публикацией

- доска открывается по внешнему HTTPS URL;
- обновление карточек работает без локального адреса;
- вложения скачиваются по защищённой ссылке;
- `СТЕНА GPT` загружается;
- bearer-токен не выключен;
- share-ссылка не раздаётся лишним людям.

## Для GPT / MCP

Публикация браузерной доски и публикация MCP — это две связанные, но разные вещи:

- для людей нужен внешний URL самой доски;
- для ChatGPT нужен отдельный внешний HTTPS MCP endpoint.

Хорошая схема:

- браузерная доска: `https://board.example`
- MCP: `https://agent.example/mcp`

## Ограничения

- сейчас `access_token` в share-ссылке фактически является секретом доступа;
- если ссылка утекла, токен нужно сменить;
- production-grade OAuth для браузерного доступа здесь пока не реализован.
