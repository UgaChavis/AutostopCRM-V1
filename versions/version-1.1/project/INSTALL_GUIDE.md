# Инструкция по portable-запуску

## Что изменилось

Для этой версии установщик больше не используется.

Теперь программа запускается как portable-выпуск:

- без мастера установки
- без записи в список установленных программ
- без ярлыка на рабочем столе
- без ручной настройки системы

## Что нужно до запуска

Перед запуском убедитесь:

1. Вы открыли папку `release`.
2. В этой папке есть файл `Start Kanban.exe`.
3. Рядом с ним есть служебная папка `_internal`.
4. Другие копии программы закрыты.

## Пошаговый запуск

### Шаг 1. Откройте папку portable-выпуска

1. Откройте папку `release`.

Что вы должны увидеть:

- `Start Kanban.exe`
- `_internal`
- `КАК ЗАПУСТИТЬ.txt`

### Шаг 2. Запустите главный файл

1. Найдите `Start Kanban.exe`.
2. Дважды щёлкните по нему.

После этого программа должна открыться без терминала.

### Шаг 3. Дождитесь открытия окна

Признаки корректного запуска:

- появилось тёмное окно программы
- видны кнопки `Новая карточка`, `Новый столбец`, `Помощь`
- отображаются стартовые столбцы
- не нужно запускать никакие дополнительные команды

## Что происходит автоматически

После запуска программа сама:

1. открывает интерфейс
2. поднимает локальный API
3. загружает сохранённые данные
4. показывает встроенную помощь при первом запуске

## Как проверить, что portable-запуск работает стабильно

### Проверка 1. Clean launch

1. Запустите `Start Kanban.exe`.
2. Убедитесь, что окно открылось сразу.

### Проверка 2. Repeat launch

1. Закройте программу.
2. Снова запустите `Start Kanban.exe`.
3. Убедитесь, что повторный запуск тоже проходит без терминала.

### Проверка 3. Close and relaunch

1. Создайте карточку.
2. Закройте окно.
3. Снова откройте программу.
4. Убедитесь, что карточка осталась на доске.

## Как удалить portable-выпуск

Отдельного uninstall нет, потому что установка в систему не выполняется.

Чтобы убрать portable-выпуск:

1. Закройте программу.
2. Удалите папку `release` или перенесите её в другое место.

Важно:

- пользовательские данные в `%APPDATA%\Minimal Kanban` при этом останутся
- это сделано специально, чтобы не терять карточки и логи

## Что делать при ошибке запуска

Проверьте по порядку:

1. Запускаете ли вы именно `Start Kanban.exe`.
2. Не удалена ли папка `_internal`.
3. Закрыты ли другие копии программы.
4. Есть ли лог-файл `%APPDATA%\Minimal Kanban\logs\minimal-kanban.log`.

Если рядом с `.exe` нет папки `_internal`, portable-выпуск неполный и его нужно собрать заново.
