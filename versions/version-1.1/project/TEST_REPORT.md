## Что было найдено и исправлено

### Исправлено

- в проект добавлен отдельный MCP adapter layer вместо встраивания логики в UI
- локальный API расширен новыми endpoint-ами `set_card_deadline`, `set_card_indicator`, `list_overdue_cards`
- добавлена опциональная bearer token защита локального API
- добавлена bearer token защита MCP server
- исправлена проблема release-скрипта: зависимости теперь обновляются даже если `.venv` уже существует
- устранена потенциальная межпроцессная гонка записи в `state.json` за счёт lock-файла

### Найдено как ограничение

Интеграционные тесты MCP проходят успешно, но Python MCP SDK при shutdown всё ещё оставляет шумные предупреждения `ResourceWarning` / `Task was destroyed but it is pending` в stderr. Это не ломает выполнение тестов и не меняет итог `28/28 OK`, но зафиксировано как внешний шум SDK/transport shutdown.

## Риски и ограничения

- полноценный OAuth provider для ChatGPT app auth ещё не реализован
- bearer token режим подходит для локальных и промежуточных интеграций, но не заменяет production OAuth flow
- для подключения с телефона требуется публичный HTTPS URL до MCP server, обычно через tunnel или reverse proxy
- шумные shutdown warning-и MCP SDK пока остаются
