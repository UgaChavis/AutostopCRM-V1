AUTOSTOP CRM SERVER AGENT WORKFLOW

Current execution model:
1. task is added to queue
2. server agent picks oldest pending task
3. agent calls OpenAI model gpt-5.4-mini
4. model selects one CRM tool at a time
5. tool result is appended to context
6. agent returns final result
7. run and actions are stored on disk

Recommended task patterns:
- board review and overload summary
- find overdue or stalled cards
- create a card from user text
- move a card to a target column
- update repair order fields
- inspect cashboxes and payment flows

Do not rely on invented ids.
Always use ids returned by tools.
